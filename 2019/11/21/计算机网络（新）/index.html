d:计算机网络协议、接口、服务等概念

基本概念

名词

  • ISP(Internet? Service? Provider)：因特网服务提供商
    • 网络由一个由多个分组交换机和多段通信链路组成。端系统（主机）通过ISP介入因特网。
  • Protocol：协议
    • 控制网络中信息接收和发送的一组软件
    • 因特网协议：TCP/IP协议
    • TCP（Transmission Control Protocol）：传输控制协议
      IP（Internet Protocol）：网际协议
  • RFC（Request For Comments）：IETF制定的标准文档。是因特网标准，由IETF（Internet Engineering Task Force）制定。
  • Intranet：专用的内部网络
    • 所用主机、路由器、链路和协议等与因特网相同。
    • 专网内的主机不能随意与专网外部的主机交换信息(由防火墙控制)。
  • 网络划分
    • 网络边缘
      • 外围部件、主机
      • 网络应用
    • 网络核心
    • 路由器
    • 通信链路
  • 端系统：与因特网相连的计算机
    • 在“网络边缘“
    • 运行应用程序，如Web、电子邮件、网络游戏等
    • 客户机（Client）：用户，桌面和移动PC和PDA等等
    • 服务器（Server）：提供服务，功能更强的机器，如Web服务器和邮件服务器

计算机网络的组成

有以下分类标准

  • 组成部分：硬件、软件、协议
  • 工作方式：边缘部分、核心部分
  • 功能组成：通信子网、资源子网

计算机网络的功能

  • 数据通信
  • 资源共享
  • 分布式处理
  • 提高可靠性
  • 负载均衡

计算机网络的分类

冒号后为关键词

按分布范围分
  • 广域网（WAN）：长距离；因特网的核心部分；高速链路；交换技术
  • 城域网 (MAN) ：以太网
  • 局域网 (LAN) ：广播技术
  • 个人区域网 (PAN)
按传输技术分
  • 广播式网络：共享公共信道；局域网、广域网中的无线、卫星通信网络。
  • 点对点网络：采用分组存储转发路由选择；广域网。

按拓扑结构分

  • 星形网络
  • 总线型网络
  • 环形网络：令牌环局域网
  • 网状形网络

按使用者分类

  • 公用网
  • 专用网

按交换技术分

  • 电路交换网络
    • 专有线路；阶段：连接、传输、断开；电话网络。
    • 数据直接传送，时延小。但线路利用率低、不能充分利用线路容量、不便进行差错控制。
  • 报文交换网络
    • 加辅助信息构成报文；即存储-转发网络
    • 可以一对多、多对一访问、可以差错控制；但是增大了资源开销，增加了缓冲时延。
  • 分组交换网络
    • 固定长度的数据块的存储-转发网络
    • 缓冲易于管理，时延小，平均缓冲区更少，易于标准化。普遍采用。

按传输介质分类

  • 有线
  • 无线

性能指标（后面介绍）

分层结构

层

  • 从低到高，第一层，第二层…
  • 不同机器上同一层为对等层/对等实体
  • n 层实现的服务为 n+1 层利用

PDU

  • 分为两部分：数据部分(SDU)， 控制信息部分(PCI)
  • 服务数据单元（SDU）：为完成用户所要求的功能而应该传送的数据。
  • 协议控制信息（PCI）：控制协议操作的信息。
  • 协议数据单元（PDU）：对等层次之间传送的数据单位称为该层的PDU。
  • 第 n 层写为：n-SDU/n-PCI/n-PDU
  • PDU的通俗名称：
    • 物理层PDU：比特
    • 链路层PDU：帧
    • 网络层PDU：分组
    • 传输层PDU：报文

协议

  • 因特网中的所有活动，都受协议制约。
  • 因特网的运行离不开协议。
  • 定义了通信实体之间发送、接收报文的格式 和传输顺序 ，以及收到报文所采取的动作。
  • 是对等实体进行通信的规则集合，是水平的。
  • 协议的构成：
    • 语法：传输数据的格式
    • 语义：所要完成的功能
    • 同步：时间实现顺序

应用层
提供各种网络应用。传输应用报文
HTTP协议，FTP协议，SMTP协议，DNS协议等

运输层
在应用程序的客户机和服务器之间提供传输应用层报文服务。传输报文段。——进程间
TCP协议，UDP协议

网络层
主机和主机之间传输网络层分组（数据报）——主机间
IP协议，ICMP协议，选路协议

数据链路层
在邻近单元之间传输数据（帧 ）
PPP，以太网协议

物理层
在节点之间传输比特流
EIA-232C、EIA/TIA RS-449、CCITT的X.21

  • 协议分层优点:
    概念化，便于了解协议的规范细节
    结构化，各层之间独立，易于实现和维护
    模块化，减少耦合度，便于更新系统组件

接口

  • 同一结点相邻两层之间交换信息的连接点
  • 同一结点相邻两层之间的实体通过**服务访问点（SAP）**进行交互
  • 第 n 层的SAP 就是第 n+1 层可以访问第 n 层服务的地方

服务

  • 下层为紧邻的上层提供功能调用。是垂直的
    • 协议是水平的
  • 服务中一些命令，称为OSI原语：
    • 请求：服务用户 $ \rightarrow $ 服务提供者
    • 指示：提供者 $ \rightarrow $ 用户
    • 相应：用户 $ \rightarrow $ 提供者
    • 证实：提供者 $ \rightarrow $ 用户
  • 服务分类：面向连接/无连接；可靠/不可靠；有应答/无应答
  • 面向连接的服务
    • 可靠的数据传送（无差错、按序）
    • 流量控制
    • 拥塞控制
  • 无连接服务
    • 不可靠
    • 无流量控制、拥塞控制

OSI 参考模型

层 传输单位 定义 功能
应用层 用户与网络的界面
表示层 处理两个通信系统中交换信息的方式 ①采用抽象的标准方法定义数据结构，并且采用标准的编码形式。②数据压缩、加密解密
会话层 不同主机间各个进程之间进行会话 建立同步（SYN），建立、终止、管理进程间的会话
传输层 报文段（TCP）
用户数据报（UDP）		 主机中两个进程之间的通信 为端到端连接提供可靠的传输服务。
流量控制，差错控制、服务质量、数据传输等服务
网络层 数据报 通信子网的运行控制 把网络层的协议数据单元（分组）从源端传到目的端，为分组交换网上的不同主机提供通信服务。
路由选择、流量控制、差错控制、拥塞控制
数据链路层 帧 将网络层传来的IP数据包组装称帧 流量控制、差错控制、传输管理
物理层 比特 在物理媒体上为数据端设备透明地传输原始比特流

TCP/IP模型

层 协议
应用层 HTTP, SMTP, DNS, FTP
传输层 TCP, UDP
网际层 IP
网络接口层
层 OSI TCP/IP
传输层 支持无连接、面向连接的通信 支持无连接、面向连接的通信
网络层 仅支持面向连接 仅支持无连接

考试大纲
a:数据链路层的功能
